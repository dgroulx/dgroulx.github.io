Intro to Object Oriented Programming

Our First Object

While iOS comes with a wide variety of existing classes to use, in most every program we will need to roll our own to describe a particular problem domain.

The Interface File (*.h)

Like functions, class definitions are split into .h files and .m files. A basic class interface looks like this: 

@interface SBSPerson : NSObject

@end

That's all it is. The @interface keyword begins a class definition. Next comes the name of our new class. A ':' then the name of our classes superclass. NSObject forms the base of the Objective-C class hierarchy. Every class we make will either directly inherit from NSObject, or will inherit from a class that inherits from NSObject. @end signifies the end of a class declaration.

The Implementation File (*.m)

#import "Person.h"

@implementation Person

@end

See how class declarations are paralleling function declarations? These two files create a fully valid class. You can now go ahead and create instances of these classes. 

#import "Person.h"

...

Person *somebody = [[Person alloc] init];
Person *otherPerson = [[Person alloc] init];

Notice that we create instances of our new class the same as we created NSArray and NSDictionary instances.

Excersizes

  1. Create the Person from this page. Make sure that you can create instances of Person and compile your program.
